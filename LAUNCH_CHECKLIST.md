# Launch Checklist

Use this before a **general product launch** (marketing, go-live, support).

## Pre-launch

- [ ] All [MVP_LAUNCH_CHECKLIST](MVP_LAUNCH_CHECKLIST.md) items completed
- [ ] All [PRODUCTION_CHECKLIST](PRODUCTION_CHECKLIST.md) items completed
- [ ] Landing page / marketing site live and accurate
- [ ] Pricing page (if applicable) and signup CTA working
- [ ] Legal pages: Privacy Policy, Terms of Service (and Cookie Policy if needed)
- [ ] Support channel defined: email, chat, or docs + FAQ
- [ ] Status page or incident communication plan (optional but recommended)
- [ ] Analytics and conversion tracking configured (e.g. signups, key actions)
- [ ] SEO: meta tags, sitemap, robots.txt (already present)
- [ ] Social / OG tags for sharing (already in index.html)
- [ ] Redirects: www → non-www (or vice versa) and HTTPS enforced
- [ ] Email deliverability: auth emails (Supabase) tested; no spam issues

## Launch day

- [ ] Final smoke test: sign up, add memory, chat, sign out
- [ ] Monitoring and alerts verified (errors, latency, uptime)
- [ ] Team on standby for first 24–48 hours
- [ ] Announcement prepared (blog, social, email)
- [ ] Rollback plan documented (e.g. revert deploy, feature flags)

## Post-launch (first week)

- [ ] Daily check of errors and key metrics
- [ ] User feedback channel monitored
- [ ] Known issues documented and prioritized
- [ ] First iteration backlog created from feedback
